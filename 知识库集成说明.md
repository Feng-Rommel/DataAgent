# 知识库检索 Agent 集成说明

## 概述

本项目已成功集成知识库检索功能，知识库检索器会根据用户的需求自动从知识库中检索相关信息，并将其提供给 QA agent 和代码生成 agent，从而提高回答的准确性和代码生成的质量。

## 架构设计

### 1. 知识库检索器 (KnowledgeRetriever)

位置：`src/knowledge_retriever.py`

主要功能：
- **自动索引**：启动时自动遍历知识库目录（`./知识库测试/`），建立分类索引
- **智能分类选择**：使用 LLM 根据用户查询自动选择相关的知识库分类
- **知识提取**：从相关文件中提取与查询最相关的知识片段
- **上下文优化**：避免检索过多的内容，保持上下文大小合理

知识库分类：
- `scRNA` - 单细胞 RNA-seq 相关知识
- `bulkRNA` - 批量 RNA-seq 相关知识
- `spatialRNA` - 空间转录组相关知识
- `datasets` - 数据库相关知识

### 2. 集成到其他 Agent

知识库检索器已集成到以下组件：

#### ProjectAdvisor (问答 Agent)
- 在回答用户问题时自动检索相关知识
- 将检索结果作为上下文提供给 LLM
- 提高回答的准确性和权威性

#### CodeProgrammer (代码生成 Agent)
- 在生成代码时检索相关的参数设置、流程说明、代码示例
- 参考知识库中的最佳实践
- 确保生成的代码符合行业标准

#### Planner (规划 Agent)
- 在制定任务规划时检索标准流程和最佳实践
- 参考知识库中的分析流程
- 生成更合理的任务拆解

## 工作流程

### QA Agent 的知识检索流程

1. 用户提出问题
2. ProjectAdvisor 调用 KnowledgeRetriever.retrieve()
3. 检索器根据查询选择相关分类
4. 读取相关分类的文件内容
5. 提取最相关的知识片段
6. 将知识片段作为上下文提供给 LLM
7. LLM 基于上下文回答用户问题

### Code Programmer 的知识检索流程

1. 用户提出代码生成需求
2. CodeProgrammer 调用 KnowledgeRetriever.retrieve()
3. 检索器根据需求选择相关分类（如 scRNA）
4. 读取相关的流程、参数、代码示例
5. 提取最相关的内容
6. 将内容作为"知识库检索结果"传递给代码生成 prompt
7. LLM 参考知识库内容生成符合最佳实践的代码

## 性能优化

- **缓存机制**：知识库索引在启动时一次性构建，避免重复索引
- **智能分类选择**：通过 LLM 快速选择相关分类，避免检索无关内容
- **内容截断**：对检索到的内容进行智能截断，保持上下文大小合理
- **异步处理**：检索过程不会阻塞主要流程

## 配置选项

### 知识库路径

默认路径：`./知识库测试/`

修改方法：
- `main.py`: 修改 `knowledge_base_path` 参数
- `ui_app.py`: 修改 `knowledge_base_path` 参数

```python
knowledge_retriever = KnowledgeRetriever(llm, knowledge_base_path="./你的知识库路径")
```

### 检索限制

- 每个分类最多读取 3 个文件
- 内容最大长度限制为 15000 字符
- 最多选择 3 个相关分类

可在 `src/knowledge_retriever.py` 中调整这些参数。

## 常见问题

### 1. 知识库路径不正确

错误信息：
```
[知识库] 警告: 知识库路径不存在: ./知识库测试
```

解决方法：
- 确认知识库路径正确
- 确保知识库文件夹存在且包含分类目录

### 2. 未检索到相关知识

可能原因：
- 查询内容与知识库内容不匹配
- 分类选择不准确
- 知识库文件格式不正确（应为 .md 文件）

### 3. 检索速度慢

优化方法：
- 减少 `max_files` 参数（默认为 3）
- 减少 `knowledge_content` 的最大长度限制（默认为 15000）
- 使用更快的 LLM 模型

## 技术细节

### 分类选择算法

使用 LLM 进行语义匹配，通过 Prompt 引导选择相关分类。

### 知识提取算法

使用 LLM 从检索到的内容中提取最相关的片段，保留：
- 定义和概述
- 参数设置
- 代码示例
- 注意事项

### 上下文管理

知识库内容作为额外的上下文字段传递给各个 agent：
- `knowledge_base`: 用于 QA、代码生成、规划
- 格式：markdown 文本
